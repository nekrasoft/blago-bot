# Telegram-бот для саммари тендерных `.doc/.docx/.xlsx/.pdf/.rar`

Бот работает в Telegram-группе: принимает `.doc`, `.docx`, `.xlsx`, `.pdf`, `.rar`, извлекает текст и отправляет короткое саммари по ключевым условиям тендера.

## Что умеет

- Отслеживает документы в группах.
- Обрабатывает `.doc`, `.docx`, `.xlsx`, `.pdf` и `.rar`.
- Извлекает текст из `.doc/.docx`, листов `.xlsx` и текстовых `.pdf`.
- Для `.rar` распаковывает архив и анализирует поддерживаемые файлы внутри (`.doc/.docx/.xlsx/.pdf`).
- Делает саммари через OpenAI API.
- Если несколько документов отправлены одним пакетом, делает одно общее саммари по всем файлам.
- Для пакета документов учитывает последнее текстовое сообщение того же пользователя перед пакетом (за последние 30 минут; приоритет сообщению со ссылкой `zakupki.gov.ru/epz/order/...`).
- В саммари отдельно подсвечивает: территориальность, тип закупки (котировка/аукцион/и т.д.), unit-экономику (цена за т, за м3, оценка цены машины 30 м3 при наличии данных).
- В саммари с приоритетом выделяет требования к исполнителю работ: регистрация, ИП/ООО, СРО, лицензии, договоры с полигонами, опыт/техника/персонал и подтверждающие документы.
- Отдельным блоком извлекает контакты заказчика/представителя (ФИО, должность, телефон, email, источник в документе), чтобы можно было быстро связаться и уточнить детали.
- Заголовки подпунктов в саммари выделяются жирным для удобного чтения.
- Пишет в stdout `message.chat.id` и `message_id` для входящих сообщений в группах.
- Если заполнен `WHITELIST_CHAT_IDS`, работает только в этих группах; в остальных пишет, что доступ запрещен, и выходит из чата.
- Делит длинный ответ на несколько сообщений (ограничение Telegram).

## Быстрый запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env`:

- `TELEGRAM_BOT_TOKEN` - токен от BotFather.
- `OPENAI_API_KEY` - API-ключ OpenAI.
- `WHITELIST_CHAT_IDS` - опционально: список разрешенных `chat_id` групп через запятую (например: `-1001234567890,-1009876543210`). Если пусто, ограничение отключено.
- `OPENAI_MODEL` - модель для саммари (по умолчанию `gpt-4.1-mini`).

Запуск:

```bash
python3 main.py
```

## Настройка Telegram

1. Создайте бота через `@BotFather`.
2. Добавьте его в нужную группу.
3. Отключите Privacy Mode у бота в `@BotFather` (`/setprivacy` -> `Disable`), чтобы бот видел документы в группе.
4. Посмотрите stdout: бот выводит `chat_id` входящих сообщений в группе.
5. При необходимости добавьте этот `chat_id` в `WHITELIST_CHAT_IDS` и перезапустите бота.

## Структура

- `main.py` - точка входа.
- `app/telegram_bot.py` - Telegram-обработчики.
- `app/docx_parser.py` - извлечение текста из `.doc/.docx/.xlsx/.pdf`.
- `app/archive_parser.py` - извлечение документов из `.rar`.
- `app/summarizer.py` - логика саммаризации.
- `app/config.py` - чтение конфигурации из `.env`.

## Важно

- Бот использует polling. Для production лучше перейти на webhook.
- Если `WHITELIST_CHAT_IDS` пустой, бот работает во всех группах.
- Для поддержки `.doc` нужен хотя бы один инструмент на сервере: `LibreOffice` (рекомендуется), `antiword` или `catdoc`.
- Для `.pdf` извлекается только текстовый слой. Для сканов нужен OCR (в проекте не включен).
- Для поддержки `.rar` нужен хотя бы один архиватор: `unrar`, `7z`, `bsdtar` или `unar`.
- На Ubuntu/Debian можно установить так:

```bash
sudo apt-get update
sudo apt-get install -y libreoffice unrar p7zip-full
```

## Частая ошибка установки

Если видите ошибку вида:

`Could not find a version that satisfies the requirement jiter<1,>=0.10.0`

значит у вас подтянулась несовместимая ветка `openai` для текущего окружения/индекса пакетов.
В проекте уже зафиксирована совместимая версия `openai<2.0.0`, поэтому просто обновите `pip` и повторите установку:

```bash
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```
